## Pré-requisitos

- [node](https://nodejs.org) - _Nós recomendamos que você tenha a última versão LTS instalada_.
- Um editor de texto, nós recomendamos [VS Code](https://code.visualstudio.com/) com a extensão [Vetur](https://marketplace.visualstudio.com/items?itemName=octref.vetur) ou [WebStorm](https://www.jetbrains.com/webstorm/)
- Um terminal, nós recomendamos utilizar o [terminal integrado](https://code.visualstudio.com/docs/editor/integrated-terminal) do VS Code ou [terminal do WebStorm](https://www.jetbrains.com/help/webstorm/terminal-emulator.html).

## Utilizando `create-nuxt-app`

Para iniciares de maneira rápida o teu projeto, podes utilizar [create-nuxt-app](https://github.com/nuxt/create-nuxt-app).

Certifica-te de que tens o yarn, npx (incluído por padrão no npm v5.2+) ou npm (v6.1+)instalado.

::code-group

```bash [Yarn]
yarn create nuxt-app <project-name>
```

```bash [NPX]
npx create-nuxt-app <project-name>
```

```bash [NPM]
npm init nuxt-app <project-name>
```

::

O terminal fará algumas perguntas (nome, opções do Nuxt, abstração de interface de utilizador, TypeScript, linter, abstração de testes, etc). Para obteres mais detalhes a respeito de todas estas opções consulte a [documentação de `create-nuxt-app`](https://github.com/nuxt/create-nuxt-app/blob/master/README.md).

Uma vez que todas as questões forem respondidas, serão instaladas todas as dependências. A próxima etapa é navegar para a pasta do projeto e executá-lo:

::code-group

```bash [Yarn]
cd <project-name>
yarn dev
```

```bash [NPM]
cd <project-name>
npm run dev
```

::

Agora a aplicação está executando sobre [http://localhost:3000](http://localhost:3000). Bom trabalho!

::alert{type="info"}
Um outra maneira de começar com a Nuxt é utilizar a [CodeSandbox](https://template.nuxtjs.org) a qual é uma excelente maneira de experimentar a Nuxt e partilhar o teu código com as outras pessoas.
::

## Instalação Manual

A criação de um projeto de Nuxt a partir do zero só exige um ficheiro e um diretório.

Nós utilizaremos o terminal para criar os diretórios e ficheiros, sinta-se a vontade para criá-los utilizando o teu editor preferido.

### Configurar o Teu Projeto

Crie um diretório vazio com nome do teu projeto e navegue para dentro dele:

```bash
mkdir <project-name>
cd <project-name>
```

_Substitua `<project-name>` com o nome do teu projeto._

Crie o ficheiro `package.json` :

```bash
touch package.json
```

Preencha o conteúdo do teu `package.json` com:

```json{}[package.json]
{
  "name": "my-app",
  "scripts": {
    "dev": "nuxt",
    "build": "nuxt build",
    "generate": "nuxt generate",
    "start": "nuxt start"
  }
}
```

O objeto `scripts` define os comandos de Nuxt que serão executados com o comando `npm run <command>` ou `yarn <command>`.

#### **O que é um ficheiro `package.json`**

O ficheiro `package.json` é como o cartão de identificação do teu projeto. Ele contém todas as dependências de projeto e muito mais. Se não souberes o que ficheiro `package.json` é, recomendamos fortemente que leias rapidamente sobre o assunto na [documentação do npm](https://docs.npmjs.com/creating-a-package-json-file).

### Instalar a Nuxt

Uma vez criado o `package.json`, adicione `nuxt` no teu projeto através de `npm` ou `yarn` como é apresentado abaixo:

::code-group

```bash [Yarn]
yarn add nuxt
```

```bash [NPM]
npm install nuxt
```

::

Este comando adicionará a `nuxt` como uma dependência ao teu projeto e o adicionará ao teu `package.json`. O diretório `node_modeles` também será criado o qual é onde todos os teus pacotes e dependências instalados estão armazenados.

::alert{type="info"}

Um ficheiro `yarn.lock` ou `package-lock.json` também é criado o qual garante uma instalação consistente e dependências compatíveis dos teus pacotes instalados no teu projeto.

::

### Criar a Tua Primeira Página

A Nuxt transforma todos os ficheiros `.vue` dentro do diretório `pages` em uma rota para a aplicação.

Criar o diretório `pages` no teu projeto:

```bash
mkdir pages
```
Depois, criar um ficheiro `index.vue` no diretório `pages`:

```bash
touch pages/index.vue
```

É importante que esta página seja chamada de `index.vue` visto que esta será página principal padrão que Nuxt mostra quando a aplicação abrir.

Abra o ficheiro `index.vue` no teu editor e adicione o seguinte conteúdo:

```html{}[pages/index.vue]
<template>
  <h1>Hello world!</h1>
</template>
```

### Iniciar O Projeto

Execute o teu projeto digitando um dos seguintes comando abaixo no teu terminal:

::code-group

```bash [Yarn]
yarn dev
```

```bash [NPM]
npm run dev
```

::

::alert{type="info"}

Nós utilizamos o comando `dev` quando estamos executando a nossa aplicação no modo de desenvolvimento.

::

Agora a aplicação está executando sobre **[http://localhost:3000](http://localhost:3000/)**.

Abra-o no teu navegador clicando a ligação no teu terminal e deves ver o texto "Hello World" que nós copiamos na etapa anterior.

::alert{type="info"}

Quando iniciado o Nuxt no modo de desenvolvimento, ele estará escutando por mudanças nos arquivos dentro dos demais diretórios, então não há necessidade de reiniciar a aplicação quando por exemplo adicionar páginas novas.

::

::alert{type="warning"}

Quando executares o comando `dev` será criado uma pasta com o nome de `.nuxt`. Essa pasta deve ser ignorada pelo controle de versão. Você ignorar ficheiros criando um ficheiro `.gitignore` na raiz do projeto e adicionando `.nuxt` ao ficheiro.

::

### Passo extra

Crie uma página com o nome de `fun.vue` no diretório `pages`.

Adicione um `<template></template>` e inclua um cabeçalho com uma frase engraçada dentro.

Então, vá para o teu navegador e veja a tua nova página em **[localhost:3000/fun](http://localhost:3000/fun)**.

::alert{type="info"}

A criação de um diretório com o nome de `more-fun` colocando um ficheiro `index.vue` dentro dele, dar-te-á o mesmo resultado tal com se criasses o ficheiro `more-fun.vue`.

::
